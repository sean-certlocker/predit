# LLM Handover Document: Predit

## 🚀 Quick Start for Future Agents
The project is a prediction market platform called **Predit**.
- **Backend**: Go (8080)
- **Frontend**: React (5173)
- **AI Referee**: Python (8000)

## 🏗 Architecture Details
1. **Market State Machine**: Defined in `backend/internal/market/service.go`. Valid transitions are critical. `DRAFT -> OPEN -> LOCKED -> RESOLUTION_PENDING -> RESOLVED/VOID -> SETTLED`.
2. **AI Webhook Flow**:
   - `Admin` triggers resolution in Frontend.
   - Backend calls `/tasks` on AI Service (Python).
   - AI Service waits (simulates window) then POSTs back to `/internal/ai/resolution` on Backend.
   - Backend updates Market status and applies safety flags.
3. **Ledger**: Currently a simple in-memory service (`backend/internal/ledger/service.go`) handling "Play Money" deposits and entries.

## 🔑 Key Constraints & Conventions
- **Token Efficiency**: Always use `rtk` (Rust Token Killer) for shell commands if available. Avoid reading large directories.
- **YOLO Mode**: Operations should be direct and autonomous.
- **Safety**: AI Resolution includes an `NSFW` and `VIOLENCE` flag check. If flagged, the market is automatically VOIDED.
